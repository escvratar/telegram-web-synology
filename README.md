# Telegram Web A на Synology NAS

Self-hosted официальный веб-клиент Telegram ([telegram-tt](https://github.com/Ajaxy/telegram-tt))
в виде Docker-контейнера на Synology NAS. Доступен по HTTPS в локальной сети.

Это **настоящий веб-клиент Telegram**, а не VNC / RDP / WebRTC-трансляция:
браузер напрямую общается с серверами Telegram по MTProto.

## Как это работает

Скрипт `setup.sh`, запущенный на NAS:

1. Проверяет Docker и Node.js (при необходимости ставит Node 22 через nvm).
2. Скачивает исходники telegram-tt с GitHub.
3. **Собирает их на хосте** (`npm run build`).
4. Генерирует самоподписанный SSL-сертификат.
5. Пакует готовый `dist/` в минимальный nginx-образ.
6. Запускает контейнер.

> **Почему сборка на хосте, а не внутри Docker?**
> На Synology сборочные Docker-контейнеры не имеют доступа в интернет
> (блокировка iptables), поэтому `npm` не может скачать зависимости внутри
> контейнера. Сборка идёт на самом NAS, где интернет есть, а в Docker
> попадают уже готовые статические файлы.

> **Почему нужен HTTPS и обратный прокси?**
> Telegram Web использует `crypto.subtle` и `SharedArrayBuffer`, которые
> браузер отключает в небезопасном контексте (обычный HTTP на IP-адресе).
> Без HTTPS приложение покажет «Your browser is not supported».
> Контейнер отдаёт обычный HTTP, а HTTPS обеспечивает встроенный в DSM
> обратный прокси с настоящим сертификатом (см. шаг 4). Так не нужен
> самоподписанный сертификат, который блокируют некоторые браузеры.

## Требования

- Synology DSM 7+
- **Container Manager** (Docker) установлен из Центра пакетов
- **SSH включён**: Панель управления → Терминал и SNMP → Включить службу SSH
- ~2 ГБ свободной RAM и ~3 ГБ места на диске для сборки
- Telegram API credentials (см. шаг 1)

## Шаг 1. Получить Telegram API credentials

1. Откройте <https://my.telegram.org/apps> и войдите по номеру телефона.
2. «API development tools» → создайте приложение.
3. Сохраните **api_id** (число) и **api_hash** (строка).

## Шаг 2. Доставить `setup.sh` на NAS

### Вариант A — Windows (`deploy.bat`)

1. Скачайте этот репозиторий.
2. Запустите `deploy.bat`.
3. При первом запуске введите IP NAS, порт SSH, имя пользователя и пароль.
   Они сохранятся в `deploy.config` и `synology.pass` рядом с батником
   (эти файлы в `.gitignore` — на GitHub не попадут).
4. Скрипт сам скачает `plink.exe`/`pscp.exe` и скопирует `setup.sh` на NAS.

### Вариант B — Mac / Linux / вручную

```bash
scp -P 22 setup.sh ВАШ_ПОЛЬЗОВАТЕЛЬ@IP_NAS:/volume1/docker/telegram/
```

## Шаг 3. Запустить установку на NAS

```bash
ssh ВАШ_ПОЛЬЗОВАТЕЛЬ@IP_NAS -p 22
bash /volume1/docker/telegram/setup.sh
```

Скрипт спросит путь установки, `api_id`, `api_hash` и порт.
Первая сборка занимает **15–30 минут** (скачивание ~900 МБ зависимостей
и компиляция). Повторные запуски быстрее — зависимости кэшируются.

## Шаг 4. Настроить обратный прокси (обязательно)

Контейнер отдаёт обычный HTTP на `http://IP_NAS:4430`, но Telegram Web
по HTTP не работает (нужен HTTPS / secure context). HTTPS обеспечивает
встроенный в DSM обратный прокси.

В DSM: **Панель управления → Портал входа → Дополнительно →
Обратный прокси-сервер → Создать**:

| Поле                | Значение                                          |
|---------------------|---------------------------------------------------|
| Источник: протокол  | HTTPS                                             |
| Источник: имя хоста | ваш домен, напр. `telegram.ваш-nas.synology.me`   |
| Источник: порт      | любой свободный, напр. `4431`                     |
| Назначение: протокол| HTTP                                              |
| Назначение: хост    | `localhost`                                       |
| Назначение: порт    | `4430` (порт контейнера из шага 3)                |

На вкладке **«Пользовательский заголовок»** нажмите **Создать → WebSocket**.

Назначьте сертификат в **Панель управления → Безопасность → Сертификат**
(можно бесплатный Let's Encrypt для адреса `*.synology.me`).

После этого открывайте Telegram по адресу обратного прокси:

```
https://telegram.ваш-nas.synology.me:4431
```

> Можно вместо порта в источнике указать поддомен — тогда адрес будет без
> номера порта. Для доступа только из локальной сети подойдёт и адрес вида
> `https://IP_NAS:4431` с сертификатом DSM.

## Управление контейнером

Все команды выполняются в каталоге установки (по умолчанию
`/volume1/docker/telegram-web`):

```bash
docker compose down       # остановить
docker compose up -d      # запустить
docker compose logs -f    # логи
```

Обновить до свежей версии telegram-tt:

```bash
rm -rf /volume1/docker/telegram-web/.build
bash /volume1/docker/telegram/setup.sh
```

## Состав репозитория

| Файл           | Назначение                                              |
|----------------|---------------------------------------------------------|
| `setup.sh`     | Установщик. Запускается на самом NAS.                   |
| `deploy.bat`   | Windows: копирует `setup.sh` на NAS по SSH.             |
| `.gitignore`   | Исключает локальные настройки и секреты.                |

Файлы `deploy.config`, `synology.pass`, `plink.exe`, `pscp.exe` создаются
локально при работе `deploy.bat` и в репозиторий не коммитятся.

## Безопасность

- HTTP-порт контейнера (`4430`) держите доступным только локально —
  наружу публикуется HTTPS-адрес обратного прокси.
- Используйте настоящий сертификат (Let's Encrypt в DSM), а не
  самоподписанный — некоторые браузеры полностью блокируют последний.
- `api_id` / `api_hash` привязаны к вашему аккаунту — не публикуйте их.

## Дисклеймер

Неофициальный проект для удобного self-hosting. Telegram Web A и торговая
марка Telegram принадлежат Telegram. Используйте на свой страх и риск.
